MULTI-WINDOW MODEL
  - Static window "main" in tauri.conf.json: small controller UI (note count, New Note).
  - Each note: WebviewWindowBuilder with label = note UUID (same as notes.id).
  - Every window loads index.html; App.tsx branches on window label.
  - main → loadNotes() → controller div
  - uuid → loadMyNote() → get_my_note matches window.label() to notes.id
  - restore_note_windows() in lib.rs recreates all note windows on app start.

TAURI COMMANDS (src-tauri/src/commands/notes.rs)
  All registered in lib.rs invoke_handler.

  get_all_notes() -> Vec<Note>
    SELECT * FROM notes ORDER BY sort_order, created_at

  get_my_note() -> Note
    Uses tauri::Window label as note id: SELECT * FROM notes WHERE id = ?

  create_note() -> Note
    1. INSERT note with UUID, defaults (color #ffeb3b, 320x400, position 200,200)
    2. Release DB lock BEFORE window creation (deadlock avoidance)
    3. WebviewWindowBuilder: decorations false, transparent true, resizable true

  update_note(
    id, title, content, plain_text, color,
    position_x, position_y, width, height,
    collapsed, float_on_top, opacity, locked
  ) -> ()
    Updates DB + set_position/set_size on matching webview window.
    Frontend invoke uses camelCase: plainText, positionX, floatOnTop, etc.

  delete_note(id) -> ()
    DELETE FROM notes; close webview window if open.

FILE RESPONSIBILITIES

    setContent(content, false) on external content sync to avoid loops.

    row_to_note() maps SQLite row → Note struct.

STATE FLOW
  1. Type in editor → debounced onChange → updateNote (optimistic) → invoke update_note → Rust persists
  2. Drag title bar → startDragging or manual position updates → update_note
  3. Resize handle → mousemove → update_note width/height
  4. New note from main → create_note → new window opens with fresh UUID
  5. App launch → restore_note_windows → each note window loads and calls get_my_note

EXTENSION RECIPES
  New Tauri command:
    1. Add #[tauri::command] in commands/
    2. Register in lib.rs invoke_handler
    3. invoke() from noteStore or component
  New note field:
    1. ALTER or extend migrations.rs
    2. Note struct Rust + TS interface
    3. row_to_note, update_note SQL + params, store.updateNote payload
  New UI in note window:
    1. Component under src/components/
    2. Wire in NoteWindow.tsx, styles in App.css

PITFALLS
  - DB Mutex deadlock if held during window create (create_note releases lock explicitly)
  - get_my_note fails if window label ≠ note id
  - Optimistic update without snapshot → cannot rollback on invoke failure
  - Editor content sync must use setContent(..., false) to prevent cursor/history bugs
  - Color picker needs stopPropagation on mouseDown or drag starts accidentally
  - float_on_top: update DB + call setAlwaysOnTop on webview
  - shapes/tags/bg_images tables exist but have no Rust commands or UI yet — do not assume they work
